    初遇SpringBoot

    简介

    Spring Boot是一个简化Spring开发的框架。用来监护spring应用开发，约定大于配置，去繁就简，just run 就能创建一个独立的，产品级的应用。

    我们在使用Spring Boot时只需要配置相应的Spring Boot就可以用所有的Spring组件，简单的说，spring boot就是整合了很多优秀的框架，不用我们自己手动的去写一堆xml配置然后进行配置。从本质上来说，Spring Boot就是Spring,它做了那些没有它你也会去做的Spring Bean配置。

    优点

    1. 快速创建

    打开浏览器，访问https://start.spring.io/，进入如下页面

    选择Project（项目类型）Language（语言）SpringBoot(SpringBoot版本)、Group（组/包名），Artifact（项目名）、Description（描述）、Package name（包名）Packaging（打包方式）、Java(Java版本)等信息点击下方“Generate”即可下载程序包。同时点击Explore可导出pom文件。也可以使用Idea快速创建。

    2. 独立运行

    • spring-boot的启动方式主要有三种:
      1. 运行带有main方法类
      这种方式很简单，我主要是通过idea的方式，进行执行。这种方式在启动的时候，会去自动加载classpath下的配置文件
    2. 通过命令行 java -jar 的方式

    执行：
    java -jar jar_path –param
    jar_path: 指代将项目打包为jar打包之后的存储路径
    –param: 为需要在命令行指定的参数。例如:
    java -jar jar_path –server.port=8081
    可指定项目启动的端口号，该命令行参数，将会覆盖application.properties中的端口配置

    3. 通过spring-boot-plugin的方式

    如果需要正常使用该maven查件，需要我们在maven项目中增加以下插件配置:

    <plugin>
  <groupId>org.springframework.boot</groupId>
  <artifactId>spring-boot-maven-plugin</artifactId>
</plugin>

    进入项目的根目录，执行:
    mvn spring-boot:run -Drun.arguments=”–server.port=8888”
    arguments：指定的参数，多个参数以”,”进行分割。

    3.主流框架集成

    例如SpringBoot集成Mybatis,在pom中增加以下依赖：

    <dependency>
    <groupId>org.mybatis.spring.boot</groupId>
    <artifactId>mybatis-spring-boot-starter</artifactId>
    <version>1.3.2</version>
</dependency>
<dependency>
    <groupId>mysql</groupId>
    <artifactId>mysql-connector-java</artifactId>
    <scope>runtime</scope>
</dependency>

    并且在配置文件中增加如下配置:

    spring:
  datasource:
    username: XXXX
    password: XXXX
    url: jdbc:mysql://localhost:3306/springboot?useUnicode=true&characterEncoding=utf-8&useSSL=true&serverTimezone=UTC
    driver-class-name: com.mysql.jdbc.Driver

mybatis:
  #Mapper.xml文件路径
  mapper-locations: classpath:mapping/*Mapper.xml
  #实体类路径
  type-aliases-package: com.xiongdiblog.entity

#showSql日志打印mysql级别
logging:
  level:
    com:
      xiongdiblog:
        mapper : debug

    最后在启动类中加上注解用于给出需要扫描的mapper文件路径@MapperScan(“com.xiongdiblog.mapper”) ，如下

    package com.xiongdiblog;

import org.mybatis.spring.annotation.MapperScan;
import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;

@MapperScan("com.xiongdiblog.mapper") //扫描的mapper
@SpringBootApplication
public class DemoApplication {

    public static void main(String[] args) {
        SpringApplication.run(DemoApplication.class, args);
    }
}

    这样就完成了SpringBoot与Mybatis的集成，相比于Spring而言，这种集成方式少了复杂的xml配置，使用起来更加简单便捷。

    4. 内嵌Servlet,可直接JAR运行

    SpringBoot中内嵌了Servlet和TomCat可以直接使用java -jar XXX.jar启动jar包。

    5. 环境区分

    在SpringBoot中可以通过配置进行配置文件的选择，SpringBoot启动时默认加载application.properties文件。可以通过spring.profiles.active=XXX来指定不同的配置文件。
    例如：

    spring:
  profiles:
    active: test

    表示启动使用application-test.properties文件中的配置。配置文件加载顺序为：启动时附带的配置–>application.properties—>application-test.properties。
    我们可以通过不同的配置来选择不痛环境的配置文件，能够兼容多环境开发。

    6. 开箱即用，不需要配置繁琐的XML

    SpringBoot不管是bean配置，连接池配置、框架集成配置等，相比较于传统的Spring配置都简单的多，完全不需要进行繁琐的XML配置，这可以说是程序员的福音。
