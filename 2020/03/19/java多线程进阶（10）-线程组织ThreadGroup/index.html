    在java中，使用ThreadGroup来组织线程，每个线程都将加入一个ThreadGroup，默认情况下新的线程都会加入到main线程所在的Group中。

    ThreadGroup就像一个线程组织，利用ThreadGroup可以对该Group下的线程做一些操作和约束，但并不能用于管理线程。

    1、ThreadGroup的父子关系

    线程有父子关系，ThreadGroup同样有父子关系，下图就很好的说明了ThreadGroup的父子关系：

    2、创建ThreadGroup

    创建线程组非常简单，可以通过下面两个构造方法来创建：

    public ThreadGroup(String name);
public ThreadGroup(ThreadGroup parent, String name);

    第一个构造函数赋予了线程组名字。第二个构造函数赋予了线程名字，同时指定了父线程组，使用示例如下：

    public class ThreadGroupCreater {
    public static void main(String[] args) {
        ThreadGroup currentGroup = currentThread().getThreadGroup();
        ThreadGroup group1 = new ThreadGroup("group1");
        ThreadGroup group2 = new ThreadGroup( group1, "group2");

        System.out.println("group1's parent is currentGroup: " + (group1.getParent() == currentGroup));

        System.out.println("group2's parent is group1: " + (group2.getParent() == group1));
    }
}

    3、复制Thread数组和ThreadGroup数组

    3.1 复制Thread数组

    如下两个方法可以将ThreadGroup中的active线程复制到Thread数组中：

    public int enumerate(Thread[] list);
public int enumerate(Thread[] list, boolean recurse);

    如果recurse为true，会将所有子group中的active线程都递归到Thread数组中。

    3.2 复制ThreadGroup数组

    和复制Thread数组类似，如下两个方法可以复制当前ThreadGroup下的子group到数组中，recurse参数控制是否递归。

    public int enumerate(ThreadGroup[] list);
public int enumerate(ThreadGroup[] list, boolean recurse);

    3.3 使用示例：

    import java.util.concurrent.TimeUnit;
import static java.lang.Thread.currentThread;
public class ThreadGroupEnumerate {
    public static void main(String[] args) {
        // 创建两个线程组
        ThreadGroup group1 = new ThreadGroup("group1");
        ThreadGroup group2 = new ThreadGroup(group1, "group2");

        // 创建一个线程加入group1
        new Thread(group1, () -> {
            while (true) {
                try {
                    TimeUnit.SECONDS.sleep(2);
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
            }
        }, "myThread").start();

        ThreadGroup mainGroup = currentThread().getThreadGroup();

        // 创建Thread数组
        Thread[] list = new Thread[mainGroup.activeCount()];

        // 复制mainGroup下所有Thread（包含子线程组中的Thread）到list
        int recurseSize = mainGroup.enumerate(list);
        System.out.println("recurse is true,recurseSize = " + recurseSize);

        // 只复制mainGroup下的Thread（不包含子线程组中的Thread，即group1中的线程不会被复制）
        recurseSize = mainGroup.enumerate(list, false);
        System.out.println("recurse is false,recurseSize = " + recurseSize);

        // 创建ThreadGroup数组
        ThreadGroup[] groupList = new ThreadGroup[mainGroup.activeGroupCount()];
        // 复制mainGroup下所有ThreadGroup（包含子线程组中的ThreadGroup）groupList
        recurseSize = mainGroup.enumerate(groupList);
        System.out.println("recurse is true,recurseGroupSize = " + recurseSize);

        recurseSize = mainGroup.enumerate(groupList, false);
        System.out.println("recurse is true,recurseGroupSize = " + recurseSize);
   }
}

    4、ThreadGroup的操作

    4.1 ThreadGroup的基本操作

    • activeCount() 用于获取Group中活跃的线程数，不能百分之百的保证数字一定正确，只是估值
    • activeGroupCount() 用于获取Group中活跃的子Group数，这也是一个近似值
    • getMaxPriority() 用于获取Group的优先级，默认为10，所有的子线程优先级不能大于group的优先级
    • getName() 获取group名称
    • getParent() 获取父group，若不存在，返回null
    • list() 该方法没有返回值，所有子线程信息会输出到控制台，即System.out
    • parentOf(ThreadGroup g) 判断当前group是不是给定group的父group。给定自身也会返回true
    • setMaxPriority() 指定group的最大优先级，最大优先级不能超过父group的最大优先级，该方法不仅会改变当前group的优先级，还会改变所有子group的优先级

    4.2 ThreadGroup的interrupt

    • interrupt() 执行一个group的interrupt方法会导致该线程组下的所有Thread都被interrupt

    4.3 ThreadGroup的destroy

    • destroy() 用于销毁一个ThreadGroup，该方法只针对没有active的线程组进行一次destroy标记，该方法会将自己在父group中移除

    4.4 守护ThreadGroup

    • setDaemon(boolean daemon) 设置ThreadGroup为守护ThreadGroup，不会影响成员线程的daemon属性，如果一个ThreadGroup的daemon为true，则当该组没有active线程时，自动destroy
